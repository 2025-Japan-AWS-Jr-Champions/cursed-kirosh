# Documentation Requirements

When working on submission documentation, record all Kiro usage details in the appropriate files under `docs/submission-requirements/`.

## Required Documentation Topics

### Vibe Coding (`vibe-coding.md`)
- Conversation structure and prompting strategies
- Most impressive code generation examples
- Iterative refinement approaches

### Agent Hooks (`agent-hooks.md`)
- Specific automated workflows
- Development process improvements
- Time savings and efficiency gains

### Spec-Driven Development (`spec-driven-development.md`)
- Spec structure and organization
- Process improvements vs vibe coding
- Incremental development benefits

### Steering Docs (`steering-docs.md`)
- Strategies for improving AI responses
- Most impactful steering configurations
- Context management approaches

### MCP (`mcp.md`)
- Extended capabilities and integrations
- Workflow improvements enabled
- Features that would be difficult without MCP

## Documentation Style

- Be specific with examples and metrics
- Show before/after comparisons where relevant
- Highlight strategic decisions and experimentation
- Demonstrate depth of understanding, not just feature usage
